# PRD (Product Requirements Document) - 학점마스터 (Grade Masters)

## 1. 제품 개요

### 제품명
학점마스터 (Grade Masters)

### 제품 설명
PDF 학습 자료를 입력받아 내용을 요약하고, 예상 문제를 생성하여 시험 준비를 돕는 AI 기반 학습 보조 챗봇 서비스

### 비전
"시험 준비의 효율을 극대화하는 AI 학습 파트너"

## 2. 목표 및 목적

### 주요 목표
- 대학생의 효과적인 시험 준비 지원
- 학습 자료의 핵심 내용 빠른 파악
- 개인화된 문제 생성 및 학습 피드백 제공

### 해결하고자 하는 문제
- 방대한 학습 자료를 짧은 시간에 이해하기 어려운 문제
- 실제 시험과 유사한 연습 문제 부족
- 취약 부분에 대한 체계적인 학습 관리 부재

## 3. 타겟 사용자

### Primary User
- **대상**: 중간고사/기말고사를 앞둔 대학생
- **특징**:
  - 시간이 부족한 상황에서 효율적인 학습 필요
  - 디지털 학습 도구에 익숙함
  - PDF 형태의 강의 자료 보유

### User Needs
- 핵심 내용 빠른 파악
- 실전과 유사한 문제 풀이 연습
- 취약점 파악 및 보완

## 4. 핵심 기능

### 4.1 PDF 요약 기능
- **설명**: 업로드된 PDF 문서의 핵심 내용을 자동 요약
- **주요 기능**:
  - PDF 파일 업로드
  - AI 기반 내용 분석 및 요약
  - 챕터별/주제별 요약 제공
  - 핵심 키워드 추출

### 4.2 문제 은행 (Question Bank)
#### 4.2.1 예상 문제 생성
- **설명**: 요약된 내용을 바탕으로 주관식 예상 문제 자동 생성
- **주요 기능**:
  - 난이도별 문제 생성
  - 문제 유형 선택 (서술형, 단답형 등)
  - 생성된 문제 저장 및 관리

#### 4.2.2 문제 출제 및 채점
- **설명**: 생성된 문제로 모의시험 진행 및 자동 채점
- **주요 기능**:
  - 문제 세트 구성
  - 답안 작성 인터페이스
  - AI 기반 자동 채점
  - 점수 및 피드백 제공

#### 4.2.3 오답노트 및 관련 내용 정리
- **설명**: 틀린 문제에 대한 상세 해설 및 관련 내용 재학습
- **주요 기능**:
  - 오답 자동 수집 및 분류
  - 상세 해설 제공
  - 관련 개념 추가 학습 자료 제공
  - 취약 분야 분석 및 시각화

## 5. 사용자 스토리

### User Story 1: PDF 요약
"대학생인 나는 50페이지 분량의 강의 자료를 빠르게 파악하고 싶어서, PDF를 업로드하면 핵심 내용을 5분 내에 요약해서 볼 수 있다."

### User Story 2: 문제 생성
"시험을 앞둔 나는 실전 연습을 하고 싶어서, 학습한 내용을 바탕으로 예상 문제를 생성받아 풀어볼 수 있다."

### User Story 3: 오답 학습
"문제를 틀린 나는 내 취약점을 보완하고 싶어서, 오답노트를 통해 관련 내용을 다시 학습할 수 있다."

## 6. MVP (Minimum Viable Product) 범위

### Phase 1 (MVP)
1. PDF 업로드 및 텍스트 추출
2. 기본 요약 기능
3. 간단한 문제 생성 (5-10문제)
4. 기본 채점 기능

### Phase 2
1. 고급 요약 (챕터별, 주제별)
2. 다양한 문제 유형 지원
3. 오답노트 기능
4. 학습 진도 관리

### Phase 3
1. 개인화된 학습 추천
2. 학습 분석 대시보드
3. 협업 학습 기능

## 7. 기술 요구사항

### Frontend (이 프로젝트의 범위)
- **Framework**: Next.js 16.0.1
- **UI Library**: React 19.2.0
- **Styling**: Tailwind CSS v4
- **TypeScript**: 엄격한 타입 체크

### Backend (별도 프로젝트)
- **Framework**: FastAPI
- **LLM Integration**: OpenAI GPT-4o, Anthropic Claude 3
- **PDF Processing**: PDF 텍스트 추출 라이브러리

## 8. 성공 지표

### 정량적 지표
- 사용자당 평균 PDF 업로드 수: 5개/월
- 생성된 문제 풀이율: 80% 이상
- 평균 세션 시간: 20분 이상

### 정성적 지표
- 사용자 만족도: 4.0/5.0 이상
- 학습 효율성 향상 체감
- 시험 성적 향상 피드백

## 9. 제약사항 및 고려사항

### 제약사항
- **범위**: 프론트엔드 개발에 한정 (백엔드는 별도 팀)
- **파일 형식**: PDF 파일만 지원 (초기 버전)
- **언어**: 한국어 우선 지원

### 고려사항
- LLM API 응답 지연에 대한 UX 처리
- 대용량 PDF 파일 처리 최적화
- 개인정보 및 학습 데이터 보안
- 모바일 반응형 디자인 필수

## 10. 타임라인

### Sprint 1 (Week 1-2)
- 프로젝트 설정 및 기본 UI 구성
- PDF 업로드 기능 구현

### Sprint 2 (Week 3-4)
- 요약 결과 표시 UI
- 문제 생성 및 표시 기능

### Sprint 3 (Week 5-6)
- 문제 풀이 인터페이스
- 채점 결과 표시

### Sprint 4 (Week 7-8)
- 오답노트 기능
- 최종 테스트 및 배포 준비
